# Project Analysis: Hybrid EdTech Frontend

## 1. Project Overview
The **Hybrid EdTech Frontend** is a modern web application built with **Next.js 16** (App Router) and **React 19**. It serves as a marketplace platform connecting students with tutors for online, offline, and hybrid learning modes. The application features multi-role support (Student, Tutor, Admin) and handles everything from discovery and searching to booking and profile management.

## 2. Technology Stack
The project leverages a cutting-edge stack for performance, scalability, and developer experience:

*   **Framework**: Next.js 16.2.4 (App Router)
*   **Library**: React 19.2.4
*   **Language**: TypeScript
*   **Styling**: 
    *   **Tailwind CSS 4**: The latest utility-first CSS framework.
    *   **shadcn/ui**: High-quality, accessible components.
    *   **Lucide React**: Modern iconography.
*   **State Management**: **Zustand** for lightweight, predictable client-side state.
*   **Data Fetching**: **Axios** with customized interceptors for handling JWT-based authentication.
*   **Form Management**: **React Hook Form** integrated with **Zod** for robust schema-based validation.
*   **Feedback**: **Sonner** for elegant toast notifications.

## 3. Project Structure
The project follows a clean, modular `src` directory layout:

*   **`src/app/`**: Next.js App Router containing all routes and page layouts.
    *   `admin/`: Admin-specific management views.
    *   `tutor/` & `tutors/`: Tutor dashboard and search discovery.
    *   `login/`, `register/`, etc.: Comprehensive authentication flow.
*   **`src/components/`**: Reusable UI components.
    *   `ui/`: shadcn/ui primitives.
    *   `Navbar.tsx`: Persistent navigation across the app.
*   **`src/services/`**: The API abstraction layer, separating business logic from UI components.
*   **`src/store/`**: Centralized state management using Zustand (e.g., `authStore`).
*   **`src/lib/`**: Configuration for external libraries like Axios.
*   **`src/types/`**: Global TypeScript definitions for consistency.

## 4. Core Architecture
### Authentication & Security
*   **JWT Pattern**: Uses tokens stored in `localStorage` for session persistence.
*   **API Interceptors**: `src/lib/axios.ts` automatically attaches the Authorization header to outgoing requests and handles `401 Unauthorized` responses globally.
*   **Route Protection**: Uses a `ProtectedRoute` component and `middleware.ts` to manage access control.

### State Management
*   Zustand provides a global store that is easily accessible across components without the boilerplate of Redux.

## 5. Key Features Analyzed
*   **Dynamic Search**: Ability to filter tutors by subject, mode, and city.
*   **Role-Based Access**: Specialized dashboards for Admins, Tutors, and Students.
*   **OTP Verification**: Integrated flow for secure user registration.
*   **Responsive Design**: Fully optimized for mobile and desktop using Tailwind CSS 4.

## 6. Recommendations
*   **Server Components**: Explore moving more data fetching into Next.js Server Components to reduce client-side bundle size and improve SEO.
*   **Role Metadata**: Consider storing user roles in cookies for better SEO-friendly redirects in Middleware.
